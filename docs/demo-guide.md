# 「第一步」演示指南

## 产品目标

这个产品解决的不是“记不住任务”，而是“用户知道该做，但很难开始”。

演示时要重点传达 3 件事：

- 录入门槛很低，只要一句话
- 提醒不是命令式催促，而是帮用户更容易开始
- 看板统计关注“启动接受率”，而不是“完成率”

## 演示前准备

### 推荐命令

```bash
npm install
npm run db:migrate
npm run db:seed
npm run dev
```

### 本地地址

- [http://localhost:3000](http://localhost:3000)

## 适合演示的页面顺序

1. 首页 `/`
2. 任务列表 `/tasks`
3. 提醒中心 `/reminders`
4. 数据看板 `/dashboard`

## 最小演示路径

### 第 1 步：新建一个任务

- 页面：`/`
- 操作：在大输入框输入“今晚把简历第一段改掉”，点击“加入任务”
- 预期结果：任务立即入库，页面显示成功反馈
- 重点观察：录入只需要一句话，其他字段都可选

### 第 2 步：在任务列表确认任务已创建

- 页面：`/tasks`
- 操作：找到刚创建的任务
- 预期结果：能看到任务内容、创建时间、场景、提醒状态和下一次提醒时间
- 重点观察：任务刷新后仍然存在，说明数据已落库

### 第 3 步：手动触发提醒

- 页面：`/tasks`
- 操作：对刚创建的任务点击“手动触发提醒”
- 预期结果：任务的下一次提醒时间会变成当前时间附近
- 重点观察：这个动作是为了让演示不需要等待真实时间推进

### 第 4 步：进入提醒中心并响应一次

- 页面：`/reminders`
- 操作：找到刚刚那条任务，点击一次“现在去做”或“10 分钟后提醒”
- 预期结果：
  - 点击“现在去做”：记录一次启动接受，并清空下一次提醒
  - 点击“10 分钟后提醒”：记录一次延后，并把下一次提醒顺延 10 分钟
- 重点观察：提醒文案偏向“先做最小一步”，不是命令式催促

### 第 5 步：进入看板查看统计变化

- 页面：`/dashboard`
- 操作：查看顶部统计卡片和下方表格
- 预期结果：
  - 有效提醒总次数来自真实 `reminder_sent`
  - 接受 / 延后 / 拒绝次数和比率会反映你刚才的操作
- 重点观察：
  - 如果刚才点了“现在去做”，启动接受次数应增加
  - 如果刚才点了“10 分钟后提醒”，延后次数应增加
  - 分组表会按提醒风格和任务场景展示差异

## 最适合演示的 seed 任务

### 1. 学习 30 分钟算法题

- 适合演示：当前应提醒任务
- 原因：seed 后它会直接出现在提醒中心，适合快速演示响应闭环

### 2. 明天下午给 HR 回消息

- 适合演示：手机场景 + gentle 风格
- 原因：可以说明“低阻力启动”不是让用户一次做完，只是先回一句消息

### 3. 周四前投 3 个岗位

- 适合演示：DDL 压力型文案 + 拒绝样本
- 原因：可以对比不同提醒风格和不同响应结果在看板中的变化

## 演示时重点观察的数据变化

- 有效提醒总次数
- 启动接受次数
- 延后次数
- 拒绝次数
- 启动接受率
- 不同提醒风格的接受率对比
- 不同任务场景的接受率对比

## 演示备注

- 如果想把数据恢复到稳定的演示状态，重新执行一次 `npm run db:seed`
- 当前版本是站内提醒模拟，不会真的发系统通知
- 看板统计来自数据库中的真实事件，不是静态 mock 数字
