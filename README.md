# 第一步 First Step

一个面向拖延型用户的**低阻力启动提醒产品 Demo**。

## 项目简介

“第一步”不是传统待办工具，也不是完整的计划管理软件。  
它聚焦解决一个更具体的问题：

> 用户明明知道自己该做什么，却很难开始。

这个项目的目标，不是帮助用户做复杂规划，而是通过**低成本录入、轻量提醒、最小行动引导**，降低从“知道要做”到“愿意开始”的阻力。

---

## 项目解决的问题

传统待办工具通常要求用户：

- 先想清楚完整计划
- 填写任务、时间、分类、优先级
- 持续维护任务列表

但对很多拖延型用户来说，真正的卡点不是“不会规划”，而是：

- 懒得录入
- 不想被命令式提醒
- 明知该做却启动困难
- 计划工具本身也会变成新的负担

因此，这个项目聚焦的是：

- **一句话快速录入任务**
- **更符合心理接受度的提醒文案**
- **围绕“先开始一下”而不是“必须完成”设计交互**
- **用“启动接受率”而不是“完成率”衡量效果**

---

## 核心功能

- 一句话录入任务
- 任务列表管理
- 轻量提醒中心
- 启动/延后/拒绝三种反馈
- 看板统计（启动接受率、延后率、拒绝率）
- 面向拖延型用户的低压文案设计

## 使用方式

### 1. 快速录入
在首页输入一句任务，例如：

- 明天下午给HR回消息
- 今晚改简历第一段
- 周四前投3个岗位

### 2. 等待提醒
系统会根据规则生成轻提醒，不要求用户做完整计划。

### 3. 对提醒做反馈
用户可以选择：

- **先开始一下**
- **等会再提醒我**
- **今天先放过它**

### 4. 查看看板
在看板页可以看到：

- 有效提醒次数
- 启动接受次数
- 延后次数
- 拒绝次数
- 启动接受率

## 适合谁使用

这个产品更适合：

- 有拖延倾向的人
- 不喜欢复杂待办软件的人
- 懒得做详细计划的人
- 需要被轻推一下才能开始的人
- 求职 / 学习 / 轻办公场景用户

###当前版本说明

当前版本为 MVP / Demo 版本，重点验证的是：

用户是否愿意录入任务
用户是否愿意接受提醒
不同提醒表达是否能降低启动阻力

暂未覆盖：

生产级数据库方案
原生移动端提醒
微信/备忘录/提醒事项深度接入
完整账号系统
长期稳定的数据持久化

###设计理念

这个项目的核心不是“提高完成率”，而是：

先帮助用户迈出第一步。

相比“做完任务”，对很多拖延型用户来说，“愿意开始”本身就是更关键的一步。
